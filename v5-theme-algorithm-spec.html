🧮 スコア計算式と 5 特徴量

全特徴量は同カテゴリ候補内の percentile 正規化 (0〜1)。外部 DB 稼働前は外部項の重みを実績へ振替 (実績 0.6) — スキーマは 5 本のまま、稼働日に有効化。
実績 ×0.4同軸過去投稿の先行指標平均 標本数補正: n<5 は (n·raw+(5−n)·0.5)/5。時間減衰 30日半減
伸び率 ×0.2直近14日 vs 前14日の比 トレンド感度。percentile 化して加算
競合 ×0.2競合出現頻度 ラベル+競合 単一パス (2026-05-08 再設計) を流用
外部 ×0.2外部トレンド DB シグナル DB 構築中 — 稼働まで重み 0 の placeholder
飽和 (減点)min(0.05×直近4サイクル採用回数, 0.15) 勝ちテーマの使い潰し防止。加重でなく減点項

📊 カテゴリ別パラメータ表 — 同一骨格・別パラメータの 3 インスタンス

根拠: Imp-CV 逆相関の実測 (2026-07-02、CV は動線系に集中) + toC カテゴリ 3 分類 lock (2026-07-03)。CV は疎すぎるため日次判定に使わない。
カテゴリ先行指標 (日次判定の物差し)探索枠 (7枠中)勝ち判定
エンタメPFV (プロフ来訪)役割 = 送客。カテゴリ判定基準と同じものを測る1同カテゴリ percentile 上位 30%
生活LC (リンククリック)役割 = プロフ導線×商材。CV の 1 段手前1同上
教育PFV (当面)商材導線設置後に LC へ切替検討2拡張期 (本数 3→16)同上

🔁 週次判定フロー — 事前固定、結果を見てから変更しない

読み方: 出力 → 計測 → 判定 → 還流 の 4 段。還流が切れた決定システムはただの静的ルール表。
CEO Run 出力スコア上位 + 探索枠 = 7 テーマ → シート書込
→
投稿・計測カテゴリ先行指標 (PFV / LC) を蓄積
→
最低条件ゲート軸ごと 3 投稿 かつ 7 日経過。未満 = 判定持ち越し
→
週次判定上位 30% = 勝ち / 下位 30% ×2 サイクル連続 = 負け
↩ 勝ち負け両方をスコア表へ還流 (探索枠の外れも「この軸×この数値」として必ず記録) ／ 月次: 先行指標×CV の相関を較正 — 崩れたら物差し自体を見直す
サイクル = 週次CEO Run に同期 3 日サイクルは PFV/LC が薄すぎ見送り (§決定ログ 6)
1 投稿判定の禁止3 投稿 7 日ミニマム 薄いデータ判定の罠 (2026-06-17 SOP) の構造的回避
判定は相対評価絶対値でなく percentile プラットフォーム全体の波に左右されない

🎲 探索枠ルール — 3 本

探索枠が無いと新テーマが構造的に入れず、環境変化に気づけない。
枠数はカテゴリ別エンタメ・生活 1 / 教育 2 拡張期カテゴリは探索多め (≈14% / ≈28%)
候補は不確実性優先標本数最少の軸から 昇順、同数なら競合頻度が高い順。ランダム不可 (UCB 的選択)
急落で探索 +1動的ルール 先行指標が 2 サイクル連続 30% 超急落 → +1、回復で戻す

📝 決定理由ログ — 6 決定 (2026-07-06)

#決定理由
1カテゴリ別目的関数Imp-CV 逆相関の実測 + カテゴリ 3 分類 lock との整合
2funnel 段割り当てカテゴリの役割と同じ段を測る。CV は疎すぎて日次判定不可
3外部 DB 込み 5 特徴量背景知識リッチ路線と揃える。重み 0 空回しで着手ブロック回避
4実績重視型の重み標本数補正つき自アカ実績が最も信頼できるシグナル。勝ち型先行の順序転換 (2026-06-16) と整合
5カテゴリ別探索枠拡張期の教育系は探索多め。探索専任アカは非拡大方針 (2026-06-12) と衝突するため不採用
6週次・3 投稿 7 日3 日サイクル (2026-05-08) は PFV/LC が薄すぎ、まぐれ判定多発の懸念で見送り

🔭 未決・拡張パス — 3 件

教育系 PFV→LC 切替商材導線の設置後に判断 先行指標を CV の 1 段手前へ詰める
3 日サイクル復帰投稿頻度・データ密度の向上時 最低条件 (3 投稿 7 日) を満たせる密度が前提
重みの学習化週次判定ログが溜まってから back-test で重みを 1 つずつ動かす手動チューニングが先
