V5 テーマ選定アルゴリズム — 設計と実走デモ

何をするアルゴリズムか: CEO Run の週次バッチで、アカウントごとに「次に投稿する 7 テーマ」を選ぶ意思決定システム。入力 = 過去投稿のテーマ別成績 + 競合 + トレンド、出力 = 今週の 7 テーマ。投稿結果が翌週のスコア表に還流し、人間の勘でやっていたテーマ選びが、毎週賢くなる計算に置き換わる。

score(テーマ軸) = 0.6×実績 + 0.2×伸び率 + 0.2×競合 − 飽和ペナルティ

Section 1 — アルゴリズムの作り方: 基本項目とそのつながり

意思決定アルゴリズムの中身は 3 部品

ソートや探索でなく「何を選ぶかを再現可能な手続きにしたもの」。どんな領域でも骨格は同じ 3 部品で、どれか 1 つが曖昧なまま作り込むと必ず手戻りする。
① 入力 (シグナル)判断材料になる数値・ラベル V5 では: テーマ別の過去成績 / 競合動向 / トレンド / 採用履歴
② 目的関数「何を最大化するか」1 本の定義 全設計の起点。曖昧だと以降すべてブレる
③ 決定ロジック入力 → 出力の変換規則 成熟 3 段階: ルール → スコアリング → 学習。V5 は段階 2 で開始

部品のつながり — 「回路」になって初めてアルゴリズム

決定して終わりではなく、結果が入力へ還流するループが本体。ループの回転速度 = アルゴリズムの成長速度。還流が切れた瞬間に学習は止まる。
入力テーマ別成績・競合・トレンド
→
スコアリングscore = Σ(特徴量×重み)
→
出力今週の 7 テーマ
→
計測・判定事前固定した物差しで勝敗
↩ 勝ち負け両方をスコア表へ還流 (翌週の「入力」になる) — この一周が 1 サイクル

育て方の定石 — 小さく始めてループ速度で育てる 4 段階

STEP 1
ルールベースで書く — 人間の勘の明文化。精度より「全判断がログに残る」ことが価値
STEP 2
評価指標を先に固定 — 物差しがない改善は全部「気がする」になる
STEP 3
仮説 → 1 変更 → 測定 — 1 サイクル 1 箇所。同時に 2 つ変えるとどちらが効いたか永久に不明
STEP 4
データが溜まってから学習ベースへ — 最初から ML が最頻出の失敗。ルール期のログが教師データになる

Section 2 — 各項目の深堀

深堀 A: スコアリング設計 — 4 工程 + 質を分ける 3 補正

基本形は「候補ごとに score = Σ(特徴量×重み) → 上位 N 採用」。ただし実務の質は足し算そのものより「補正」で決まる。
工程1: 特徴量列挙判断に効く要素を出す 実績 / 伸び率 / 競合頻度 / 採用履歴…
工程2: 正規化全て 0〜1 に揃える 桁の大きい項の支配を防ぐ。定番 = percentile 化
工程3: 重み付け直感を数字にする 合計 1 にすると調整しやすい
工程4: 出力規則上位 N + 探索枠 N 個中 1〜2 個はスコア無視の新テーマ枠
補正1: 標本数補正少数サンプルは平均側へ縮小 1 本 10 万再生 < 10 本平均 5 万。まぐれに釣られない
補正2: 時間減衰古い実績ほど軽く 30 日半減で開始。トレンドが速いほど速く
補正3: 飽和ペナルティ連投テーマに減点 勝ちテーマの使い潰しを防ぐ

深堀 B: 評価指標の決め方 — 4 原則

1. 北極星は 1 本最大化する指標を 1 つに絞る 下げてはいけない指標は「ガードレール枠」に分離
2. 先行指標で回す遅行指標で答え合わせ CV は遅い → 速い相関指標で日次判定、月次で較正
3. 判定ルール事前固定何投稿・何日・何超えで勝ちか 結果を見てから基準を動かした実験は実験でない
4. グッドハートの法則指標が目標になると歪む 再生数を追うと釣り化する等。ガードレール = 歪み検知装置

深堀 C: 探索と活用の配分 — 4 ルール

勝ちパターンだけ回すと学習が止まる。一定割合は意図的に外し球を打つのが普遍原則。
1. 固定割合の探索枠採用枠の 10〜20% 環境変化が速いショート動画系は高め (20% 前後)
2. 不確実性の大きい候補へランダムより情報価値 データが薄く実力不明の軸を優先 (UCB の発想)
3. 失敗も必ず還流「試して外れた」を記録 記録しないと同じ外れ球を何度も打つ
4. 配分は状況で動かす安定 → 下げる / 急落 → 上げる 環境変化検知時は探索を増やす

#決めたこと採用案理由 (1 行)
1目的関数の構造カテゴリ別 3 インスタンス (エンタメ/生活/教育)、北極星 CV は共通Imp-CV 逆相関の実測 + カテゴリ 3 分類 lock と整合
2先行指標 (日次の物差し)funnel 段割り当て: エンタメ=プロフ来訪 / 生活=リンククリック / 教育=当面プロフ来訪カテゴリの役割と同じ段を測る。CV は疎すぎて日次判定不可
3特徴量セット外部 DB 込み 5 本 (稼働まで重み 0 空回し)背景知識リッチ路線と整合、着手はブロックしない
4重みの初期値実績重視型 (実績 0.4 / 伸び 0.2 / 競合 0.2 / 外部 0.2)補正済み自アカ実績が最も信頼できるシグナル。勝ち型先行の順序と整合
5探索・活用の配分カテゴリ別: エンタメ・生活 = 7 中 1 探索 / 教育 (拡張期) = 7 中 2。急落 2 サイクル連続で +1拡張期ほど探索を厚く。探索専任アカは非拡大方針と衝突するため不採用
6判定ルールとサイクル週次・最低 3 投稿 7 日。勝ち = 同カテゴリ上位 30% / 負け = 下位 30% × 2 連続1 投稿判定 = まぐれ判定。薄いデータ判定の罠を構造的に回避

STEP 4: 7 枠への配分 — 今週このアカが投稿すべきテーマ

活用 6 枠 (スコア順に比例配分)遊郭×2 ／ 男色×2 ／ 夜這い×1 ／ 大奥×1 上位 2 軸に 2 枠ずつ、3〜4 位に 1 枠ずつ
探索 1 枠 (スコア無視)新軸「江戸の食」など実績ゼロの軸 スコアでは永遠に選ばれない軸に、意図的に 1 枠
来週の判定 (事前固定)3 投稿 7 日 → 上位 30% で勝ち 結果は勝ち負け両方 STEP 1 の表へ還流
このアルゴリズムが人間の選定と違う判断をした点 (実データで見える価値) ① 大奥を 6 連投 → 1 枠に絞った: 成績 4 位なのに直近最多投稿だった軸を、飽和ペナルティ満額 (−0.15) が自動で「一旦休ませる」判定に。 ② 男色を 2 枠に引き上げた: 5/14 の 97 万再生 (5/4 の同種投稿の 17 倍) を伸び率項が捉えた。ただし標本数補正で「まだ 3 本」の慎重さも同時に効いている。 ③ 夜這いを切らさなかった: 直近 2 週投稿ゼロで人間なら忘れる軸だが、平均 25 万の実績が枠を確保。 ④ 美容を 0 枠にした: 直近 3 連投していたが最下位。下位 30% を 2 サイクル連続なら来週さらに減点。
