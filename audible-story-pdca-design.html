🧭 4層ファネル — 指標・扱い・レバー

左から右へユーザーが流れる。層ごとに「扱い」と判定速度が違うのがこの設計の核。PDCAが回るのは完走層とタップ層のみ。
視聴層 監視のみ 指標: 週次 ΣF1 reach 不調=打ち手でなくアカ×商材の不適合。改善しないが、全実験の分母なので監視は必須
→
完走層 週次判定 指標: F1→F2残存率(主) / 完走率 hook_type文字量引きframe数Reel連動
→
タップ層 2-3週累積 指標: 型別累積 tap率 リード文型CTA型リンク箱書影
→
CV層 遅行・月次 指標: acc×月 CV book選定期待調整 CVRは測定不能 → 設計制約で代替
⚠ タップ層の罠: フリクション軽減で軽いタップを増やすと tap率↑でも CVR↓。「本を読みたくなる心理が先、軽減は後」を生成時の lint にする。

⚖️ 層別判定ルール 4層

baseline = そのアカの直近4週 reach 加重値。初期値: 完走率 0.60 / 0.58 / 0.65 / 0.62 (アカ別実測中央値)。
視聴層 監視のみ 疲弊フラグ: 4週中央値比 −30% が2週連続 母数除外: 2週中央値 F1 reach <300 → 実験対象外 フラグ時は完走層分析に「差がノイズに沈む」と注記
完走層 週次 判定: F1→F2残存率が baseline ±4pt 超で ↗/↘ n: 週4 story 未満 or ΣF1 reach <1,000 → 継続観測 週に動かす変数は1つだけ
タップ層 累積 結論: 2型とも Σreach ≥4,000 かつ tap率差2倍以上 未達: 「継続観測」と明記して持ち越し 有望 0.9% / 勝ち宣言 2.0%
CV層 月次 急落時: ①軽いタップ罠 ②bookミスマッチの順に検査 lint: 心理が先・軽減は後 / F最終で「無料で1冊」明示 book別は定性判定と明記

🔁 週次 Run の型 7ステップ

0〜5 は週次、6 は隔週。出力は weekly ファイル1枚に3点固定: scoreboard / 前週仮説の答え合わせ / 今週の律速と1変数。
0 準備
JOIN + ガード打ち手台帳 × Performance DB を story_key で結合。未収集率 >30% の週はスキップと明記。重複行は posted_at で dedup
1 視聴
監視のみΣF1 reach 推移 → 疲弊フラグ / 実験母数除外の2判定だけ
2 完走
週次判定F1→F2残存カーブ → 前週仮説の答え合わせ → 勝ち筋を design に固定 → 今週の1変数を決定
3 タップ
累積プール更新型別に reach/tap を積み、結論ルール到達分のみ判定。未達は継続観測
4 CV
月次確認CVR急落の検査と設計制約 lint の遵守確認
5 裁定
scoreboard + 律速宣言前週比で最も落ちた層を今週の律速として1つ宣言 → 実験変数1つ → design 生成へ反映
6 適合
隔週・learn アカのみ8週レバーを回し切って累積 tap率 0.9% 未達かつ CV=0 → 停止 or 商材替えを提案
