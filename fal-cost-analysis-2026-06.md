## 4. 方法論・注記

**データ取得**
- Fal REST API `GET api.fal.ai/v1/models/requests/by-endpoint`（`expand=payloads`）で6月全19 endpoint × 全ページを取得。15,153レコード / ユニーク13,706件（openai×fal エイリアス重複を除外）。
- Fal はリクエストに「アカウント」情報を保持しない（Auth Method別も空）。**アカウント紐付けはプロンプトのテーマ/キャラ名による推定**。
- 生成数 = 生成工場マスター（Sheet2）の6月行。実投稿数 = performance_summary「Acc based Summary_26/06」実測。

**コスト按分ロジック**
- 各 endpoint の正確な総コスト（Fal Usage値）を、アカウントの課金量シェアで按分。合計はUsageに厳密一致（$3,952.94 ≈ $3,953.52、差は表示丸め）。
- 動画系: 課金量プロキシ = 入力動画秒数（status 200のみ）。画像系: 成功リクエスト数。
- テーマ語のない汎用カメラ指示文（例: "Mist drifts"）は同一endpoint内の分類済み構成比で按分。

**信頼度**
- 高: kling o3=97%歴史系 / gpt-image=99%歴史系 / nano-banana=93%江戸系 とテーマが明確一致。
- 中〜低: hailuo/pixverse の汎用モーション文は按分（仮定含む）。江戸4アカウントと選択系は画像から個別分離困難のため合算。

**推定値の限界**
- cost/本はグループ内一律、投稿数は割当(scheduled)で近似。歴史系 $48.7/本 が効くため、精緻化には歴史系「製作42本の実Falリクエスト単位の紐付け」が必要。
- 放棄83本にFal画像編集(gpt-image)が部分的に走っていれば、実ロスは$569より膨らむ可能性あり（要精査）。

**円換算**: ¥155/$ で概算。実績「70万円」認識は実効 ≈ ¥177/$ 相当で、差はFX・カード手数料・消費税等と推定。
