Algorithm Design Guide

アルゴリズムの作り方

ここで言うアルゴリズムはソートや探索でなく意思決定システム — 「何を選ぶか」を再現可能な手続きにしたもの。中身は世界共通で3部品しかなく、育て方の定石は「小さく始めて、ループの速さで育てる」。実例はショート動画のテーマ選定で通す。

🧩 中身は3部品 — どんな意思決定アルゴリズムも同じ骨格

どれか1つが曖昧なまま他を作り込むと必ず手戻りする。特に目的関数が最優先。
① 入力 (シグナル)判断材料になる数値・ラベル 例: 過去投稿の視聴データ / ジャンルラベル / 競合動向 / 経過日数
② 目的関数「何を最大化するか」1本の定義 例: 視聴数か、CVか、フォロワー転換か。ここが全設計の起点
③ 決定ロジック入力 → 出力の変換規則 成熟度で3段階: ルール → スコアリング → 学習ベース
段階1: ルールベースif-then の明文化 人間の勘をそのまま規則に。精度よりログが残ることが価値
段階2: スコアリング要素 × 重みの足し算 score = Σ(特徴量×重み)。手動チューニングで運用
段階3: 学習ベース重みをデータが決める 十分なデータが溜まってから。最初からやるのは禁じ手

🔁 全体ループ — アルゴリズムは「回路」で完成する

決定して終わりではなく、結果が入力に還流して初めてアルゴリズム。還流が切れた瞬間に学習は止まる。
入力実績・トレンド・競合
→
決定ロジックスコアリング + 探索枠
→
出力今週のテーマ 7個
→
計測事前固定した物差しで判定
↩ 結果 (当たり / 外れの両方) をスコア表へ還流 — このループの回転速度が成長速度

🌱 育て方 4段階 — 小さく始めてループ速度で育てる

順序が命。最初から学習ベース (ML) に行くのが最頻出の失敗。
STEP 1
ルールベースで書く
人間の勘の明文化だけでよい。「全判断がログに残る」状態を最速で作るのが目的。
STEP 2
評価指標を先に固定する
変更の良し悪しを測る物差しがないと、以降の改善は全部「気がする」になる。
STEP 3
仮説 → 1変更 → 測定 のサイクルを回す
1回に変えるのは1箇所。同時に2つ変えるとどちらが効いたか永久に分からない。サイクルを週1→3日に縮める投資は精度改善より効く。
STEP 4
データが溜まってからスコアリング → 学習へ置換
ルールで蓄えたログが、次の段階の教師データになる。

🎯 スコアリング設計 — 4工程 + 質を分ける3補正

基本形: 候補ごとに score = Σ(特徴量 × 重み) → 上位N採用。ただし実務の質は足し算より「補正」で決まる。
工程1: 特徴量の列挙判断に効きそうな要素を出す 同ラベル平均視聴 / 直近伸び率 / 競合出現頻度 / 経過日数
工程2: 正規化全て 0〜1 に揃える 桁が大きい項の支配を防ぐ。定番 = percentile 化 (外れ値に強い)
工程3: 重み付け直感を数字にする 例: 実績.4 / 伸び率.3 / 競合.2 / 鮮度.1 — 合計1で調整しやすく
工程4: 出力規則上位N or 閾値 + 探索枠 N個中1〜2個はスコア無視の新テーマ枠を必ず確保
補正1: 標本数補正少数サンプルは平均側へ縮小 1投稿10万再生 < 10投稿平均5万。まぐれ当たりに釣られない
補正2: 時間減衰古い実績ほど軽くする 例: 30日で半減。トレンド変化が速い環境ほど減衰も速く
補正3: 飽和ペナルティ連投テーマにマイナス項 直近採用回数を減点。勝ちテーマの使い潰しを防ぐ
検証 = back-test過去データで「このスコアなら何を選んでいたか」を再現し、実際の当たりを選べていたかを見る。重みを1つずつ動かして順位変化を観察すれば手動でも詰められる。

🧮 計算実例 — テーマ4候補から今週の3枠を選ぶ

重み: 実績 0.4 / 伸び率 0.3 / 競合頻度 0.2 / 鮮度 0.1。特徴量は percentile 正規化済み (0〜1)。
候補テーマ実績 ×.4伸び率 ×.3競合 ×.2鮮度 ×.1素点標本数補正後判定
偉人の食文化2投稿・平均8.4万再生 0.900.900.600.600.810.70実績0.9→0.63に縮小 (2投稿のみ)採用 ①
偉人の意外な日常12投稿・平均5.2万再生 0.600.500.900.200.590.59補正なし (標本十分)採用 ②
都市伝説解説9投稿・平均3.8万再生 0.300.200.301.000.340.34見送り
昭和レトロ (新規)データなし スコア対象外 — 実績ゼロの候補はスコアでは永遠に選ばれない—採用 ③ 探索枠

読みどころ: (1) 首位「食文化」は補正で 0.81→0.70 に下がるが順位は維持 — 投稿が増えて実力なら信頼が回復し、まぐれなら自然に落ちる。(2) 3枠目はスコア無視の探索枠 — これが無いと新テーマが永遠に入れない。(3) 外れても「昭和レトロ×この数値」という情報が残り、次回のスコア表が賢くなる。

📏 評価指標の決め方 — 4原則

物差しが曖昧なままの改善は全部「事後正当化」になる。実験の前に決めて書き残す。
1. 北極星は1本最大化する指標は1つに絞る 下げてはいけない指標は「ガードレール枠」に分離 — 監視のみ・最適化しない
2. 先行指標で回す遅行指標で答え合わせ CVは遅すぎる → 相関する速い指標で日々判定し、月次で物差し自体を較正
3. 判定ルール事前固定何投稿・何日・何超えで勝ちか 結果を見てから基準を動かした実験は、実験ではない
4. グッドハートの法則指標が目標になると歪む 再生数を追うと釣り動画化する等。ガードレールは歪み検知装置

🎲 探索と活用の配分 — 4ルール

勝ちパターンだけ回すと学習が止まる。一定割合は意図的に外し球を打つのが普遍原則。
1. 固定割合の探索枠採用枠の 10〜20% ε-greedy。環境変化が速いショート動画系は高め (20%前後) が定石
2. 不確実性の大きい候補へランダムより情報価値 データが少なく実力不明なテーマを優先 (UCBの発想)。同じ1枠の情報量が最大に
3. 失敗も必ず還流「試して外れた」を記録 記録しないと同じ外れ球を何度も打つ。外れ = 次回のスコア表の材料
4. 配分は状況で動かす安定→下げる / 急落→上げる 複数アカ並走なら「探索多めアカ / 活用特化アカ」の役割分担も取れる

🚫 禁じ手 6 — やった瞬間に育たなくなる

最初からMLデータ蓄積前に学習ベース 教師データが無い。ルール→スコアの順で貯めてから
目的関数が複数全部最適化しようとする 全指標が中途半端に。北極星1本 + ガードレール分離
同時に2箇所変更どれが効いたか不明に 1サイクル1変更。速く回すことで本数を稼ぐ
事後に基準変更結果を見てから勝ち定義を動かす 全実験が事後正当化になる。判定ルールは事前固定
探索枠ゼロスコア上位だけ回し続ける 新テーマが構造的に入れず、環境変化に気づけない
還流の切断結果がスコア表に戻らない ループが切れた決定システムはただの静的ルール表
