💡 売れるであろうアングル案 — 10本

但し書き: 本セクションは Claude の仮説をそのまま流用したアイデア出しです。CVR・View の数値含め未実証を含みます。実行前に、各アングルの詳細の詰め (ペルソナ・競合・カタログ適合の検証) と戦略的な検討が必要です。
共通の選定原理 (実測由来): CV は Imp でなく intent から出る (6月実測: Imp-CV 逆相関)。「コンテンツの続き = Audible」のゼロ距離接続が成立するほど上位。AI faceless の構造制約 (「人柄」型は不利、「世界観」型が正) を各案に織り込み済み。
① ビジネス書要約・耳活 主軸候補 積読サラリーマン (28-38・通勤40-60分)。「その積読、耳なら通勤で月3冊」。要約で興味喚起 → 全編は耳で、の最短導線。Audible 最強カタログ領域。
② 怪談・朗読 対抗候補 深夜の怪談リスナー (20-40)。音声で物語を聴く習慣が形成済み。「この話の完全版 (プロ朗読1時間) を無料で」。江戸の闇の当たり型・資産を転用可。
③ 話題小説・ミステリー紹介 仮説 読書に戻りたい30代女性。ドラマ化・映画化の原作需要。「あの話題作、家事しながら聴けます」。※漫画は Audible に無いため対象外。
④ 歴史・時代小説 実測あり 現行江戸クラスタの隣接展開。雑学 Reels → 時代小説・歴史書の Audible へ。唯一 CV 実績のある接続 (0223/0193/0190)。
⑤ 朝活・習慣術 仮説 自己変革願望層 (20-30代)。「朝の30分、スマホを見る代わりに聴く」。習慣化系ベストセラーと接続。①の姉妹型で playbook 共用余地。
⑥ お金・投資リテラシー 仮説 NISA・家計見直し層 (25-45)。マネー本要約 → 「通勤中にお金の勉強」。金融ジャンルは注力実績あり・商材ミックス (金融系高単価案件) への布石にもなる。
⑦ 睡眠・寝ながら聴く 仮説 不眠気味の20-30代。「スリープタイマーで眠れる朗読」。リラックス映像 × AI 生成の相性◎。intent が登録まで届くかが検証点。
⑧ 子育てママの「自分時間」 仮説・要注意 30代ママ。「家事・寝かしつけ後の耳読書」+ 絵本/育児書カタログ。※ママ系は「人柄」型で AI 構造制約に最も抵触しやすい — 世界観型で組めるかが条件。
⑨ 英語・学び直し 仮説・弱め キャリア焦燥層 (25-35)。洋書多読・語学コンテンツ。intent は高いが専用アプリ勢と競合し、Audible が最適解になりにくい。
⑩ 名作文学・大人の教養 仮説 教養志向の30-50代。「読んだふりしてきたあの名作を、耳で完走」。教養層には実在ソースで濃く、の既存原則と整合。④と playbook 共用余地。
優先順位の目安 (Claude 仮説): ①② を先行2アングルとして実証 → ③⑤⑥ を第2波 → ④は現行継続 → ⑦⑧⑨⑩ は第2波の結果を見て取捨。 同一 playbook 内はテーマをずらす (同テーマ2本目急減衰の実測)。

🧱 前提条件と想定課題 — 土台11 × 問題14 × API制限

このオペの固有リスク型: 単体アカ運用では「起きても1体の損」だった問題が、playbook×API×量産では全て相関故障 (10体〜全艦に同時波及) に変わる。対応の共通形 = 工程への標準組み込み + 週次判定への canary 指標追加。
A. 土台になっている前提条件 これが立っている限りで体制・数字の絵が成立する。未検証のまま scale すると相関故障の母体になる。
前提内容状態
市場分析機能アングル決定後、人間がアカ収集 → Claude が分析・マッピングしコンテンツの土台化未整備 (これから作る)
勝てる設計書の型外してはいけない要素のテンプレ化半分 (江戸系に原型・新アングル未検証)
Story の API 投稿20日/月 配信の自動化半分 (方針確定・大規模未検証)
Reels の API 日次投稿本体投稿の自動化。リーチ不利・凍結誘発がないこと半分 (一部実績・大規模未検証)
計測パイプラインCV のアカ帰属 + funnel 4点 (View/F1/CTR/CV) の自動取得半分 (既知の計測穴あり・要修繕)

C. Meta API のアカウント数制限 (アプリ層 = BAN と別系統の単一障害点) アプリ凍結 = 全接続アカの同時投稿停止。開発モードはロール保有アカのみ (実質数十体)、本番化後は明示上限よりレート制限とアプリ信頼スコアが実効制限。現アプリのモード・実測接続上限の確認が設計の前提。
選択肢位置づけ評価
① 複数アプリへのシャーディング本命50-70体/アプリで分散。スケールと凍結リスク対策を同時解決
② 増量申請 (App Review・本番化)①の前提工程必須。ただし「1アプリに全量」はやらない
③ 外部投稿ツール溢れバッファ実コスト月 ¥1-2k/acc = 利益の15-30%。TOS リスクあり・つなぎまで
④ 手動投稿非常用のみ人時モデルが崩壊するため不採用。上位アカの障害時 fallback
