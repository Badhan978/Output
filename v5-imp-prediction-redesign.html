なぜ当たらないか — 実装調査で確定した7つの構造欠陥

#欠陥実装上の事実
1予測式が実質存在しないPlanner への指示は「過去views平均値付近で設定」の自然言語1段落のみplanner-system-prompt.md Step2。回帰・補正・レンジ設計なし
2過去実績が未供給だった実績データ注入は 2026-07-06 実装。測定対象バッチは実績ゼロの勘で一律 10,000/15,000実績なし時のデフォルト = views:1000 固定
3アカウントパワー未考慮フォロワー数は agent 定義に記載あるが未実装。ACC実力は中央値で 152〜45,014 = 297倍の幅この幅を無視して全ACC同じロジックで予測
4PFアルゴリズム知識が届かない知識は v5-agents/knowledge/ に存在 (1,527行) だが、Planner はクリーン環境で外部Read禁止 = 物理的に参照不能CEO経由の間接圧縮のみ、予測数値には非連動
5伸び要因が予測に非連動Hook/Body/CTA 分解はシナリオ生成専用。予測数値には一切入力されない結果 = テーマ弁別力 ρ −0.01
6予実照合ループ不在予測誤差を回収して次に返す機構が3リポともゼロ。駿冴シートが初の照合実測は「次のテーマ選定」にしか還流しない
7二重予測 + 過大化圧CEO「推測view」と Planner predicted_kpis が非統合。実験経路は「過去中央値を必ず超える予測を出せ」と指示goal と forecast が混線 → 構造的に過大予測

🔄 理想の予測ループ — 機械が土台、LLMは乗率だけ、毎週採点

読み方: 予測生成 (左) → 投稿・計測 (中) → 週次学習 (右) → 破線がパラメータと知識の還流。機械層 と LLM層 を分離するのが設計の核 — LLMの寄与が単独で採点可能になる。

設計原則 3本

乗率解雇ルール弁別力ρが4週連続 ≤0 なら M=1.0 固定にフォールバックLLM乗率は「価値を証明できた時だけ」稼働する honest gate
goal と forecast の分離「目標を必ず超える予測を出せ」系の注入は全撤廃目標は目標、予測は予測。CEO推測view は乗率+根拠形式に一本化
バズは当てずに検出する3倍超バズは15% — 点予測の対象外。P90帯で表現し、48h初速がP90超えでアラート予測の仕事は「通常レンジを当てる」+「異常を早く検知」

🏗️ 事前準備 (Phase 0) — 過去データでの初期学習と土台づくり

ループを回し始める前に済ませる5本。①が全ての前提。②〜④は既存505件からの「事前学習」= 初日から較正済みの状態で開始できる。
#準備内容状態
1予実パイプライン恒常化駿冴シートの突合ロジック (permalink主キー) を週次ジョブ化。マスターに予測の構造化列 (P10/P50/P90/乗率/根拠) を新設散文の仮説Docから予測を独立させる — 採点可能性の担保新設
2パラメータ・バックフィル505件で C_a・σ_a・r_a を初期化。本設計の初期値 (σ=0.66, r=1.26 等) がその算出結果= 過去データでの事前学習。初日から較正済みで開始算出済み
3コールドスタート prior 表ジャンル×PF の実測中央値表。実測3本未満の新規ACCはここから開始 (σ広め=0.85)新規アカ設計 (NAD) の期待値設定にも流用可能新設
4乗率 few-shot bank 初期版3倍超の上振れ64件・大幅下振れ事例を Agent3 が要因言語化 → 乗率プロンプトの実証例セットに「何起因で伸びたか」の教師データを過去分から先に作る新設
5KPI合意 + ガードレール撤去「予測的中率」= Hit@2x で定義合意 (7/3 経営会議のKPI軸に接続)。goalMedianViews 撤廃・CEO推測view 一本化定義が曖昧なままだと「的中率75%」等の目標が測定不能合意事項

🗓️ 実装フェーズ 0-3 — 各段で精度が測定可能に上がる

順序が効く: ベースライン (P1) が入って初めて乗率 (P3) の寄与が測れる。P3 は「挑戦」— 乗率解雇ルールで価値を毎週検証しながら進める。
Phase 0
事前準備 — 上記5本。予実ログの構造化が全ての前提。新設
Phase 1
ベースライン注入 — Planner Step2 改修: B_a·C_a を機械注入、LLM出力は乗率+根拠のみに変更。効果はバックテスト実証済み 26% → 52%。効果実証済み
Phase 2
較正ループ稼働 — 週次採点ジョブ + C/σ/r 自動更新 + P10-P90 レンジ運用開始。Coverage80 と Bias が監視下に入る。新設
Phase 3
弁別力への挑戦 — PF知識・Hook実証値・競合anchor・few-shot bank を乗率判断に注入し、ρ −0.01 → 0.3+ を狙う。乗率解雇ルール (4週連続ρ≤0でM=1固定) で価値を毎週検証。LLM層

KPI定義の注意 (7/8 前に握る)

「精度75-80%」は定義次第2倍以内定義では天井60-65%で届かない。3倍以内 (Hit@3x) なら初月70%・3ヶ月で72-75%に到達対外KPIにするならどちらの定義かを先に合意する
